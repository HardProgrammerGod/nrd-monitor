# Newly Registered Domains (NRD) Threat Intelligence Dashboard

Автоматизированная аналитическая панель для мониторинга, анализа и категоризации недавно зарегистрированных доменов (NRD). Проект разработан для автоматического сбора списков потенциально опасных хостов (фишинг, спам, C&C-инфраструктура) и их оперативной верификации через легковесный веб-интерфейс.

## 🚀 Архитектура и технологии

* **Backend:** Python 3.11+, FastAPI (асинхронный высокопроизводительный веб-фреймворк), Uvicorn (ASGI-сервер), Pydantic v2 (валидация данных).
* **Frontend:** HTML5, Modern JavaScript (Async/Await Fetch API), Tailwind CSS (JIT-компиляция через CDN), Chart.js (интерактивная визуализация метрик).
* **Security Context:** ETL-процесс (Extract, Transform, Load) для парсинга OSINT NRD-фидов, нормализация сырых строк, дедупликация и фильтрация по зонам риска (TLD).

## 📡 Спецификация API (Endpoints)

Интерактивная документация Swagger доступна автоматически после запуска бэкенда по адресу: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

* **GET /api/v1/stats** — Получение агрегированных метрик для главного дашборда
* **GET /api/v1/tlds** — Сводный список доменных зон, отсортированных по уровню угрозы
* **GET /api/v1/search** — Проверка конкретного домена на благонадежность по query-параметру ?domain=

## 🛠 Инструкция по локальному развертыванию

### 1. Запуск Backend-сервера

Перейдите в директорию бэкенда, инициализируйте окружение, установите зависимости и запустите ASGI-сервер:

cd backend
pip install -r requirements.txt
uvicorn main:app --reload

Интерфейс API успешно инициализируется на [http://127.0.0.1:8000](http://127.0.0.1:8000)

### 2. Запуск Frontend-интерфейса

Поскольку клиентская часть реализована как SPA без жестких внешних зависимостей сборщиков (Webpack/Vite), для запуска достаточно открыть файл frontend/index.html в любом современном веб-браузере.

Клиент автоматически выполнит асинхронные запросы к локальному REST API для отрисовки графиков и дашборда.
